# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 작업 워크플로우

모든 코드 수정은 아래 흐름을 따른다.

### 1. 계획 (Plan)
- Plan Mode 진입 후 구현 계획 수립
- 다관점 분석: 목적을 여러 관점(성능, 유지보수성, 확장성, 단순성 등)에서 바라보고, 각 관점에서 방법을 구상한 뒤 합의를 통해 최적 방법 도출
- GPT 전문가에게 크로스 피드백 요청 (Architect/Plan Reviewer 등 적절한 전문가 활용)
- 사용자 승인 후 구현 시작

### 2. 구현 (Implement)
- **메인 세션은 계획 검토와 진행 관리에 집중**하고, 실제 코드 수정은 서브에이전트에 위임한다.
- 승인된 계획의 독립적인 작업 단위를 식별하고, **가능한 한 여러 서브에이전트를 병렬로 소환**하여 동시 수행한다.
- 서브에이전트 소환 시 반드시 포함할 내용:
  - 수정 대상 파일 경로와 변경 내용의 구체적 명세
  - 관련 컨텍스트 (기존 코드 패턴, 재사용할 함수/유틸리티)
  - 변경 범위의 경계 (무엇을 하지 말아야 하는지)
- 서브에이전트 결과를 메인 세션에서 검토하고, 파일 간 정합성을 확인한다.
- 계획에서 벗어나는 변경이 필요하면 사용자에게 먼저 확인

### 3. 검증 (Verify)
- 계획대로 구현되었는지 항목별 체크
- 놓친 요구사항 점검
- 사이드이펙트 및 기존 기능 영향 확인
- 테스트 실행으로 동작 검증

### 4. 에러 이력 관리 (ERR_HIST/)
- 테스트 중 같은 종류의 에러가 **2회 이상** 발생하면 `ERR_HIST/` 폴더에 기록
- 파일명 형식: `YYYY-MM-DD_HHmm_{에러_요약}.md`
- 내용: 에러 내용, 원인 분석, 해결 방법, 재발 방지 포인트
- 구현 전 `ERR_HIST/` 폴더를 참조하여 동일 실수 방지

## 의사결정 기록 (DECISIONS/)

다관점 분석 후 내린 기술적 결정은 `DECISIONS/` 폴더에 기록한다.

- 파일명 형식: `YYYY-MM-DD_{결정_주제}.md`
- 내용: 배경, 검토한 선택지들, 선택 이유, 트레이드오프
- 이미 기록된 결정은 재논의하지 않는다 (변경 필요 시 새 결정으로 기록)

## 컨텍스트 인수인계

복잡한 작업이 세션을 넘어갈 때, 세션 종료 전 현재 상태를 기록한다.

- 기록 위치: `HANDOFF.md` (프로젝트 루트)
- 내용: 현재 진행 상태, 완료 항목, 남은 작업, 주의사항
- 새 세션 시작 시 `HANDOFF.md`가 있으면 먼저 읽고 이어서 작업
- 작업 완료 후 `HANDOFF.md` 삭제
